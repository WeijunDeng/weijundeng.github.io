Git文件实时同步 for Markdown

发表于 2016-04-23   |   分类于 Develop   |  

对于一个经常使用Markdown写作的程序员来说，这实在让人兴奋不已。

最终目的

你在某个文件夹下的文件变动，可以实时自动提交推送到Git。

这就是本文最终的目的。

作为程序员的你一定使用过Git或SVN，你应该可以理解什么意思。

而如果你刚好跟我一样，经常使用Markdown写作，你应该可以理解，这意味着什么：

  • 本地编辑Markdown文件，自由地记笔记或写博客，无论使用什么编辑器
  • 轻量的云存储。配置私有仓库就可以私密存储
  • 完美的历史版本记录、恢复和对比
  • 部分Git托管网站支持Markdown渲染，比如Github和Coding，这意味着，你可以在其他设备阅读Markdown渲染后的效果。比如我用Coding的App在移动端阅读我的文档。
  • 如果你使用Hexo之类的静态博客，也可以避免无法云存储你的Markdown源文件的尴尬
  • 理论上也可以多人协作编辑的，可能需要手动处理冲突

基本原理

通过工具监听某个文件夹的文件变动事件，包括增删改，设定一定的间隔，执行一段Git脚本，将改动提交推送到Git远端。

回顾与展望

Markdown的编辑器已经多如牛毛，很多已经走上了或开源或收费的道路，彼此的同质化情况也较为严重。在各个开发者不断增加功能和完善体验的更新中，我希望使用Markdown记笔记或写博客时，通过给用户开放配置Git来实现同步和协作，也可以成为一个选择。

我不会告诉你，我已经不用fswatch了，一边码字一边留意终端有没有上传，也是很碍眼又费神的。我现在是隔一段时间就手动执行一下commit.sh，反正就是3次击键的操作。所以还是App原生集成配置Git最好。
